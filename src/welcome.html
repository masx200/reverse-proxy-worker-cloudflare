如何使用 DoH 服务

将以下地址填写到你的客户端或操作系统 DNS 设置中，即可启用加密 DNS

DNS over HTTPS — GET
https://<your-worker>.workers.dev/dns-query?dns=<base64url-encoded-query>
DNS over HTTPS — POST
POST https://<your-worker>.workers.dev/dns-query
Content-Type: application/dns-message
在 Firefox 中启用
设置 → 常规 → 网络设置 → 启用基于 HTTPS 的 DNS → 自定义 → 填入上方地址
在 Windows 11 中启用
设置 → 网络和 Internet → 以太网/Wi-Fi → DNS 服务器分配 → 手动 → 加密首选 → 填入上方地址

工作原理

四步实现边缘 DNS 缓存加速

接收请求

Worker 接收来自客户端的 DoH GET 或 POST 请求，解析 DNS 查询报文。

规范化为 GET

POST 请求的二进制 body 被 Base64url 编码后附加为 ?dns= 参数，统一转换为可缓存的 GET 形式。

边缘缓存查询

借助 Cloudflare Cache API（cacheEverything: true），命中缓存的请求直接在边缘节点响应，无需回源。

负载均衡回源

缓存未命中时，从配置的多个上游 DoH 节点中随机选择并重试，全部失败时返回 502。

常见问题

快速解答你的疑惑

DoH 和传统 DNS 有什么区别？

传统 DNS 使用明文 UDP/TCP（端口 53）传输，查询内容在网络层完全可见，容易被运营商或中间人劫持。DNS over HTTPS 将查询封装在标准 HTTPS 请求中，具备加密传输和完整性校验，有效防止窃听与污染。

如何配置多个上游 DoH 服务器？

在 Wrangler 的环境变量 DOH_ENDPOINT 中填写 JSON 数组即可，例如：["https://doh.pub/dns-query","https://dns.alidns.com/dns-query"]。Worker 会随机打乱顺序后逐一尝试，直到获得有效响应。

缓存的 TTL 是多少？

缓存时长完全遵循上游 DoH 服务器返回的 DNS TTL 值（体现在 HTTP 响应的 Cache-Control 头中），Cloudflare 边缘节点会据此自动管理缓存生命周期，Worker 本身不另行干预。

如何更改监听路径？

修改环境变量 DOH_PATHNAME 即可，默认值为 /dns-query。本地开发时在项目根目录的 .dev.vars 文件中覆盖此变量，无需修改 wrangler.toml。

是否支持 DNS over TLS（DoT）？

本项目专注于 DoH（DNS over HTTPS），不支持 DoT（端口 853）。Cloudflare Workers 本身也仅暴露 HTTP(S) 端口，若需 DoT 支持，请考虑在自有服务器上部署专用代理。

部署后如何验证服务正常？

可使用 curl 测试：curl -H "accept: application/dns-json" "https://<your-worker>.workers.dev/dns-query?name=example.com&type=A"，或直接访问根路径 / 查看本欢迎页面。
